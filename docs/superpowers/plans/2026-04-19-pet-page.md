# Pet Feature Page Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a `Pet` page on the docs site (under Features) with one hero
plus three state captures (Idle / Walking / Working), driven by three new
scripted scenarios in the existing capture pipeline.

**Architecture:** Three new `fixtures/scenarios/pet-*` shell scripts
manipulate `@pane_status` to drive the pet through each state, capture a
single cropped frame, and feed `scripts/build-assets.sh`. The MDX page imports
the resulting PNGs from `astro:assets` and is wired into the Starlight
`Features` group via `astro.config.mjs`. A new `enable_pet` helper in
`fixtures/scenarios/common/_lib.sh` keeps each scenario small.

**Tech Stack:** Bash (scenario scripts), tmux (scripted layout), Rust binary
(`tmux-agent-sidebar capture`), Node + Playwright + sharp (HTML → PNG via
`scripts/render-frames.mjs`), Astro Starlight (docs site).

**Spec:** [`docs/superpowers/specs/2026-04-19-pet-page-design.md`](../specs/2026-04-19-pet-page-design.md)

### Task 2: `pet-idle` scenario

**Files:**
- Create: `fixtures/scenarios/pet-idle/scenario.sh`

Pet stays at home when no agent is running. `build_layout` seeds
`MAIN_PANE` and `PANE_RUNNING_2` as `running` — overriding both to
`waiting` post-`build_layout` makes `running_count == 0` once the next
1 s `refresh_interval` (`src/app.rs:46`) picks up the change.

- [ ] **Step 1: Create the scenario file**

```bash
mkdir -p fixtures/scenarios/pet-idle
```

Write `fixtures/scenarios/pet-idle/scenario.sh`:

```bash
#!/usr/bin/env bash
# Single-frame snapshot of the pet at its home position. Forces the
# count of running agents to zero so the cat stays at PET_HOME_X = 1.
#
# The cat may briefly take a few right-walking steps before the next
# refresh_interval (1 s, src/app.rs) picks up the @pane_status overrides
# and running_count drops to 0 — at which point the cat WalkLefts back
# home. ~2 s of extra sleep past start_sidebar's built-in 2 s gives the
# round trip plenty of headroom before capture.
#
# Usage: scenario.sh <output-dir> [extra capture args...]

set -euo pipefail

OUT="${1:?usage: scenario.sh <output-dir> [extra capture args...]}"
shift
EXTRA_ARGS=("$@")

source "$(cd "$(dirname "$0")/../common" && pwd)/_lib.sh"

# Crop to the pet scene band: 5 rows immediately above the bottom
# panel. Layout (src/ui/mod.rs:60-78) splits the canvas into pane list
# (Min 1) | pet band (Length 5) | bottom panel (Length 20). At
# canvas height 46 (build_layout), PET_SCENE_HEIGHT = 5, and
# default BOTTOM_PANEL_HEIGHT = 20, the band is rows 21..25 inclusive.
# CROP_ROWS is end-exclusive.
export CROP_ROWS=21:26
export CROP_COLS=0:46

setup "pet-idle"
trap cleanup EXIT

mkdir -p "$OUT"

build_layout

# Force running_count to 0 by overriding the two `running` panes.
# Only @pane_status matters for the pet — @pane_attention /
# @pane_wait_reason set by _seed_pane don't influence tick_pet.
tmux set-option -t "$MAIN_PANE"     -p @pane_status waiting
tmux set-option -t "$PANE_RUNNING_2" -p @pane_status waiting

enable_pet
start_sidebar

# 3 s headroom: up to 1 s for refresh_interval to pick up the override
# + ~2 s for WalkLeft to bring the cat home from pet_x ≈ 20.
sleep 3

capture_single
```

### Task 7: Create the Pet feature page

**Files:**

Use the `notifications.mdx` and `agent-pane.mdx` formats as the model:

- [ ] **Step 1: Write the page**

Create `website/src/content/docs/features/pet.mdx`:

````mdx
---
title: Pet
description: A small cat that lives at the bottom of the sidebar and animates as agents run.
---

import { Image } from 'astro:assets';
import working from '../../../assets/captures/pet-working.png';
import idle from '../../../assets/captures/pet-idle.png';
import walking from '../../../assets/captures/pet-walking.png';

<Image src={working} alt="pet working at the desk" densities={[1.5, 2]} />

A small ASCII cat lives in a 5-row band between the agent list and the bottom panel. Its state is driven by the running-agent count — the same count surfaced in the sidebar's status filter row. The pet is opt-in: enable it with `set -g @sidebar_pet on` (see [tmux options](/tmux-agent-sidebar/reference/tmux-options/)).

## States

### Idle

<Image src={idle} alt="pet sitting idle at home" densities={[1.5, 2]} />

When no agent is running, the cat sits at the left edge of the band. Periodically it plays a subtle motion — a blink, a wave, or a small jump — seeded fresh each cycle so the timing isn't perfectly regular.

### Walking

<Image src={walking} alt="pet walking across the band" densities={[1.5, 2]} />

When the running-agent count crosses 0 → ≥1 (or back), the cat walks between its home position and the desk. The walk uses mirrored sprites for both directions; the docs treat both as one Walking state.

### Working

<Image src={working} alt="pet working at the desk" densities={[1.5, 2]} />

While at least one agent is running, the cat sits at the desk shuffling papers. The paper stack grows with the running-agent count: 1 paper at 1 running agent, 2 papers at 2-3, capped at 2 for 4 or more.
````
